## Обновления после базового коммита

### Telegram: доски и задачи из Supabase
- Команда/кнопка `boards` в Telegram теперь запрашивает список досок из базы данных.
- Вместо текстового списка бот отправляет inline-кнопки с названиями досок (без отображения `id`).
- При выборе доски бот получает задачи из `tasks` по `board_id` и отправляет их пользователю списком.
- Добавлена обработка `callback_query` и подтверждение выбора через `answerCallbackQuery`.

### Обновления логики Worker
- Расширена модель `TelegramUpdate` для поддержки callback-событий.
- Унифицирована обработка Telegram-апдейтов для webhook и polling.
- В polling добавлена подписка на `allowed_updates: ["message", "callback_query"]`.

### Тесты
- Обновлены моки Supabase: добавлена поддержка выборки задач по доске.
- Переписан Telegram polling тест под новый сценарий:
  - команда `boards`;
  - показ кнопок досок;
  - выбор доски;
  - получение списка задач.

## Новые обновления (создание задач через Telegram)

### Telegram: отдельная кнопка `создание задач`
- В reply-клавиатуру добавлена новая кнопка `создание задач` рядом с `boards`.
- Реализован пошаговый сценарий:
  - пользователь нажимает `создание задач`;
  - бот просит прислать текст большой задачи;
  - после текста бот предлагает выбрать доску через inline-кнопки;
  - после выбора доски бот создает и сохраняет подзадачи в `tasks`.
- Для callback добавлен отдельный префикс `create_tasks_board:` (отдельно от `board:` для просмотра задач).

### Переиспользование логики `/chat`
- Логика разбиения задачи, выбора исполнителей и сохранения в Supabase вынесена в общую функцию.
- `POST /chat` теперь использует эту общую функцию без изменения внешнего API-контракта.
- Telegram-сценарий `создание задач` использует ту же бизнес-логику, что и `/chat`.

### Состояние Telegram-диалога
- Добавлено хранение состояния сценария в `TELEGRAM_STATE` KV:
  - `awaiting_task_text` — ожидание текста задачи;
  - `awaiting_board_selection` — ожидание выбора доски;
  - `idle` — отсутствие активного сценария.
- В `wrangler.jsonc` добавлен binding:
  - `kv_namespaces` -> `TELEGRAM_STATE` с рабочим `id`.

### Тесты
- Обновлены моки для проверки фактических insert-данных в `tasks`.
- Добавлен тест Telegram polling для полного сценария:
  - `создание задач` -> текст -> выбор доски -> создание подзадач.
- Сохранен и подтвержден безрегрессионный сценарий `boards`.

### Деплой и инфраструктура
- Выполнен деплой с подключенным KV binding `TELEGRAM_STATE`.
